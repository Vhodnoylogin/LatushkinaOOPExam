Задание на окончание модуля "Основы ООП"
Разработать сервис приложение на базе spring boot framework, которое должно выполнять следующие функции: 
 1. Прием файла comtrade по Rest api, Парсинг (разбор) файла на отдельные точки измерений мгновенных значений токов и сохранение каждой из них в базу данных (H2)
 2. Прием запроса по Rest api с параметрами (точка начала и точка конца - номера выборок) на анализ выбранного участка на наличие КЗ, и выдачу ответа по Rest api фаз, в которых есть КЗ

Приложение должно состоять из следующих модулей :
 1. Controller - rest controller с методами : 
 ⁃ Прием измерений , Post method , который в теле запроса принимает MultipartFile - файл comtrade и передает его на уровень Service, метод ничего не возвращает  
 ⁃ Приём команды на определение КЗ в загруженном файле на выбранном диапазоне, get запрос, принимает два параметра : начальная точка int и конечная точка int. Передает эти данные на уровень Service, возвращает string с 
типом КЗ(наименование фаз, в которых обнаружено превышение уставки) или пустую строку , если его нет 
 ⁃ Задание уставки по фазному току, превышение которого считается за возникновение КЗ, get запрос, принимает один параметр: double - значение тока уставки. Передает значение на уровень сервиса
 2. Service - три метода:
 ⁃ прием файла от Rest контроллера , разбивает его по точкам и поочередно передает все точки на уровень repository 
 ⁃ Принимает две точки от Rest Controller в int и запрашивает с данными параметрами из репозитория массив точек в соответствии с заданным временным промежутком, преобразует значения данного массива точек из мгновенных значений токов 
в фазах в действующие и анализирует на наличие КЗ, возвращает String с определеным типом КЗ. За определение КЗ считать превышение тока уставки
 ⁃ Принимает значение тока КЗ в double и сохраняет в переменную. данная переменная используется для вычисления проверки на КЗ. 
 3. Repository - два метода
 ⁃ добавление точки в БД, принимает объект измерения , ничего не возвращает 
 ⁃ Выбрать все точки за заданный промежуток, принимает два int значения, возвращает List из измерений
 4. Класс измерений с аннотациями для hibernate, который создаст таблицу в БД H2 и и будет использоваться репозиторием

 Проврека будет осуществлятся автоматическим тестированием собранного Jar файла, получаемоего с помощью команд mvn build.

endpoints:
